Then, what are the new technologies applied by Radya Labs?

Technologies applied by Radya Labs

1. Technology Stack
There are four application platforms that Radya Labs is working on, that are mobile, web, chatbot, and cross platform. Each platform has its own technology stack, such as the Android Application that uses Java and Kotlin or the cross platform using Xamarin, React-Native, and Flutter.

2. Development Tools
We also consider the use of the right tools to help the application development process being worked on. Currently, we are using Bitbucket for repository with the implementation of Git Flow to produce better source code management and code quality. Apart from Bitbucket, Radya Labs also uses other tools such as Visual Studio and Visual Studio Code for different purposes and uses.

3. Server
There are several servers that Radya Labs prepares for the development process according to the requirements and architecture that client need. Some of the servers used by us are Internal Server (on Premise), VPS (as staging server), and Azure Cloud Service. Each type of server has a different function, for example, such as a VPS Server which will be used if the project already in deliverable phase and ready to test by client or in our ways called as ‘weekly build’.

4. DevOps
Radya Labs has tried to implement DevOps Technology from one years ago, tools that we use is Azure DevOps. One of our goals in implementing this technology is to automate the developer's recurring jobs, such as passing object releases to the development, staging, and production servers. So, with the use of this automation, developers are expected to be able to focus more on the tasks and the generated code.

5. Code Analysis
Code analysis is used to assess the source code produced by developers to achieve maximum quality. Right now, Radya Labs already implement code analysis method by using SonarQube as a tool.

6. DevOps x Regression Test
At the end of 2019, we also doing research about the integration of DevOps UI with Regression Test, we use Katalon Studio and SmartBear as the Testing Tools. The aim of this integration is to ensure all functional applications run well before being deployed to the Production server.
